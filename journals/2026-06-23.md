# heuristicas pra cabeça
desacoplar features intensivas via Feature Flags: sempre que mexer em lógicas core (como feed e busca), proteja as novas features com Feature Flags. Isso permite colocar o código em produção silenciosamente, ligar/desligar de forma isolada, e testar na infra real sem impacto ao usuário final.

ao indexar tabelas grandes em produção, sempre use CREATE INDEX CONCURRENTLY e rode isso fora de transações (no Supabase, adicionando -- supabase: no-transaction). O banco construirá o índice em background sem travar a aplicação.

migrations não são apenas o "resultado final", mas um passo a passo. Garanta que cada migration intermediária deixe o banco num estado totalmente funcional. Se perceber que fez uma gambiarra numa migration recente que ainda não foi pra produção, consolide-a logo ou apague-a em vez de criar uma nova migration de "correção".
